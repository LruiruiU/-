# 目标：找到黑客的入侵点、了解黑客入侵后做了什么

## 开始：

* 打开 vulntarget-n 虚拟机显示出登陆界面
* 输入 账户：root 密码：Vulntarget@123
* 开启靶场
* 输入 ifconfig 查看靶机ip
* 在浏览器输入靶机ip可以看到确实被勒索加密了
* 接下来使用Xshell和Xftp连接靶机

# 找到黑客入侵点：

* 第一步、查看页面挂马情况

1. 进入Xftp，在靶机的文件中找到 **index.jsp** 配置文件进行查看 该靶机路径为“/opt/tomcat/webapps/ROOT/index.jsp”
2. 在查看index.jsp的途中看到有许多文件的后缀为 .vulntarget 被加锁了

## 查看此次危机的范围：

* 使用

```
find / -name *vulntarget*
```

* 全盘搜索文件名中带有 vulntarget 的文件
* 查询出

```
/opt/tomcat/webapps/ROOT/index.jsp.vulntarget #被修改的主页配置文件
/opt/tomcat/webapps/ROOT/.vulntarget #在ROOT目录下有一个 .vulntarget 隐藏文件怎么想都很可疑
/opt/tomcat/webapps/ROOT/vulntarget.jsp.vulntarget #不清楚
/opt/tomcat/webapps/ROOT/404.jsp.vulntarget #不清楚
/opt/tomcat/webapps/ROOT/flag.jsp.vulntarget #flag嘛，通关凭证
#通过以上文件路径还能确定中间件是 Apache Tomcat
```

* 五个 vulntarget 文件

## 继续查找入侵点：

* 输入

```
netstat -ano
```

* 查看开放的端口
* 但是并没有什么可疑的端口
* 鉴于被修改的界面是可以访问到的web页面
* 所以有理由怀疑攻击者是进行了webshell的获取，并进行了提权后将其勒索的一个步骤

## 查寻攻击手法：

* 既然怀疑是webshell获得权限那么我们可以看看访问日志
* 在上面了解到了中间件是 Apache Tomcat ，那么我们就查看一下Tomcat的默认日志存放路径
* Tomcat日志存放在 /opt/tomcat/logs 中
* 查看日期 2024.06.04(案发当天) 的访问日志
* 开始进行日志审计(这日志眼睛看着真的花)
* 先大概溜一圈看看能不能发现一些异常点
* 定眼一瞧就发现了很可疑的东西啊

# 现在我们要知道黑客入侵后做了什么：

* 输入 **history** 看看当前root用户执行过的历史命令
* 看到了一条

```
flag{vulntarget_very_G00d}
```

* 但我们还没有结束
* 又发现看了一条

```
cp web.xml web.xml.bak
#黑客把存在漏洞的 web.xml 备份了一边防止漏洞被修复他进不来了
cp index.jsp index.jsp.bak
#这个也备份了一份
crontab -e
#这里看到了一条计划任务的命令
```

* 使用 crontab -l 去看看黑客有没有创建计划任务

```
@reboot /opt/tomcat/bin/startup.sh
#确实是有一个计划任务在这，每当设备重启就会执行
```

* 通过查看发现这是黑客用于权限维持的手段，到最后都要一并删除的

```
#哦~又有重大发现
mv vulntargetn.jsp 404.jsp
#黑客把 vulntargetn.jsp 重命名为了 404.jsp
mkdir .vulntarget
#又创建了一个 .vulntarget
cd .vulntarget/
ls
mkdir keys
#在 .vulntarget/ 里面创建了一个key
python3 get_pem.py 
cat pubkey.pem 
cat privkey.pem 
#把公钥私钥放在了这上面
pip3 install rsa
#回头看黑客在这里装了一个 rsa 加密模块
python3 encrypt_vulntarget.py 
#这里就是执行了一个加密的动作
```

* 那么就可以理出来了

```
mv vulntargetn.jsp 404.jsp
#黑客把木马重命名了之后
mkdir .vulntarget
#创建了隐藏文件夹
pip3 install rsa
装载了加密模块
python3 get_pem.py 
#生成了钥匙对
python3 encrypt_vulntarget.py 
进行加密
```

### 注意：靶场是把公钥私钥留在了目标上，现实中黑客99.99%不会把密钥留下

* 既然靶场给了我们公钥私钥，也知道对方是怎么加密的，那么我们就可以尝试进行解密的一个动作了
* 还记得上面找到的五个被加密文件吗

```
/opt/tomcat/webapps/ROOT/index.jsp.vulntarget 
/opt/tomcat/webapps/ROOT/.vulntarget 
/opt/tomcat/webapps/ROOT/vulntarget.jsp.vulntarget 
/opt/tomcat/webapps/ROOT/404.jsp.vulntarget #木马也给加密了
/opt/tomcat/webapps/ROOT/flag.jsp.vulntarget
```

* 我们就先来解flag吧，通关第一位(

```
/opt/tomcat/webapps/ROOT/.vulntarget/keys
#这是公钥私钥放置的位置
```

* 找个在线解密网站就好了

```
flag{https://github.com/crow821/vulntarget}
#这就是接出来的flag了
#这里面的网址是靶场作者的github项目，里面还有很多靶场
```

* 再来看看 vulntarget.jsp.vulntarget 这个很像木马的东西是什么

```
hack by crow
#黑客：哈哈！你掉进陷阱了！
```

* 后面的解密就不一一展示了
* 要是遇到密文太长解不出来的，整个脚本解吧
